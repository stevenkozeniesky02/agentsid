# The 251 gap: the MCP servers our own registry couldn't see

We published a census of 15,983 Model Context Protocol servers last month. Every number in that census was keyed off npm and PyPI — the two public registries where most MCP code actually lives. That was the right starting point. It wasn't the whole picture.

This week we went looking for what we missed. A targeted GitHub crawl surfaced **6,633 unique repositories** plausibly related to MCP. Of those, 371 are confirmed MCP servers by manifest dependency — all absent from our registry.

**251 of them are published on npm or PyPI right now.** Our crawler just didn't find them. The other 114 are GitHub-only: installed by `git clone` plus a manual build step, never touching a public package registry.

This post is about both groups, in that order, because the 251 gap is the one we can fix first.

## The 251: coverage bugs we owe you

Our registry builder resolves packages against npm + PyPI manifests it discovers through a set of seed queries. It missed these — some of them with very large footprints:

| Stars | Repo | Package |
|---|---|---|
| 26,511 | assafelovic/gpt-researcher | `gpt-researcher` (PyPI) |
| 14,408 | GLips/Figma-Context-MCP | `figma-developer-mcp` (npm) |
| 11,811 | tadata-org/fastapi_mcp | `fastapi-mcp` (PyPI) |
| 8,443 | LaurieWired/GhidraMCP | `GhidraMCP` (PyPI) |
| 7,829 | Upsonic/Upsonic | `upsonic` (PyPI) |
| 6,661 | grab/cursor-talk-to-figma-mcp | `cursor-talk-to-figma-mcp` (npm) |
| 5,234 | getsentry/XcodeBuildMCP | `xcodebuildmcp` (npm) |
| 4,963 | sooperset/mcp-atlassian | `mcp-atlassian` (PyPI) |

That's eight high-visibility packages our users would reasonably expect to see graded. The full list of 251 is in [`scanner/docs/census-2026/github-repo-scan.json`](https://github.com/AgentsID-dev/agentsid/blob/master/scanner/docs/census-2026/github-repo-scan.json) under `published_missing`. We've opened a backfill task against the registry crawler — all 251 should be graded and live on [agentsid.dev/registry](https://agentsid.dev/registry) within the week.

We don't think this is a fatal flaw. We do think it's worth saying out loud that any grading system keyed off a package registry is only as complete as the registry's own discovery — including ours. If you operate another MCP directory, you likely have a version of the same blind spot. We'd rather find it and publish it than pretend it isn't there.

## The 114: servers no registry can see

The second group is more interesting. 114 confirmed MCP servers distribute entirely by GitHub clone — no npm `publish`, no PyPI upload, no registry entry anywhere. The canonical install path is:

```
git clone https://github.com/<owner>/<repo>
cd <repo> && pip install -e .
```

Or the Node equivalent. These servers are real, deployed, and in some cases attached to serious audiences — they just live outside the visibility of every existing registry-based grade.

We extracted tool definitions from the 55 repos where static analysis could reliably recover them, then graded them with the same v0.3.0 scanner rules used on our published-tool census. The distribution:

| Grade | Count | Share |
|---|---|---|
| A | 0 | 0% |
| B | 4 | 7% |
| C | 12 | 22% |
| D | 17 | 31% |
| F | 22 | 40% |

**71% graded D or F.** That's close to the distribution in the published-tool census. In other words: the servers that skip npm and PyPI are not systematically safer. They are not a secret cache of better-written code. They are, on average, the same code quality as the published ecosystem, just without the accountability loop.

### Four examples worth naming

**[0x4m4/hexstrike-ai](https://github.com/0x4m4/hexstrike-ai)** — 8,149 stars. An MCP server whose stated purpose is to let AI agents run offensive-security tooling: nmap, sqlmap, metasploit, and the rest. 150 extracted tools, 142 HIGH findings, grade F with a score of 15. This is not a bug. It is the product. The question it forces is governance: should an agent's default tool context include unauthenticated wrappers around active-exploitation tools?

**[microsoft/lets-learn-mcp-python](https://github.com/microsoft/lets-learn-mcp-python)** — 1,052 stars, branded `lets-learn`, hosted in Microsoft's GitHub org. Grade F. Eight tools with permission and validation findings. This is the template a lot of tutorials will start from. When the reference example grades F, the servers built on top of it start in a hole.

**[mark3labs/mcp-go](https://github.com/mark3labs/mcp-go)** — 8,600 stars. The de-facto Go SDK for MCP. The example server bundled with the framework grades F (score 33). SDK example servers set norms — they are the first real code most developers read in a new protocol. A grade-F default is a grade-F ecosystem default.

**[coleam00/remote-mcp-server-with-auth](https://github.com/coleam00/remote-mcp-server-with-auth)** — 292 stars, named for its headline feature. Grade F with two CRITICAL findings. A server that places "with-auth" in its name is setting a specific expectation; the scan says that expectation isn't met. We flag it because naming conventions are how developers find reference implementations, and a "with-auth" reference that doesn't pass an authorization audit is worse than one that makes no claim at all.

We are flagging these not to name-and-shame. We are flagging them because the gap between "published to a registry" and "well-governed" is the whole thesis of this work, and these are good evidence for it.

## What this changes

- **For our registry.** We're shipping the 251-package backfill. We're also opening a standing issue on how the registry crawler discovers packages, because "seed queries plus manifest traversal" is clearly not enough.
- **For the AgentsID Action.** Our GitHub Action runs a live scan of an MCP server on each PR and fails CI on configurable grade thresholds. We're adding a `--github-only` mode that scans repo-resident tool definitions directly, so CI can gate servers that never hit a package registry at all.
- **For anyone else running a grade or directory service.** The 251 list is in the scan JSON. Use it to sanity-check your own coverage. If you find another 251 we missed, tell us. We'll publish a diff.
